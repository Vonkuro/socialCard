
Voici où sont rangées les ressources d'un composant SocialCard :
app
    composant
            socialCard
                SocialCard.js
                socialCard.json
                defaultCard
                    defaultBackground.png
                    defaultLogo.png
                    defaultXxx.xxx

SocialCard.js contient la définition du composant avec la personnalisation par défaut
La personnalisation par défaut est compilée dans le bundle
Les personnalistions utilisateurs sont sauvegardées dans socialCard.json

Voici le format de socialCard.json :
{
    "Card": {
        "ADEO":{
            "backgroundUri": "./test",
            "logoUri": "./testLogo",
            "titleText": "titre",
            "subtitleText": "sous titre"
        },
        "Yoti":{
            "backgroundUri": "./test",
            "logoUri": "./testLogo",
            "titleText": "titre",
            "subtitleText": "sous titre"
        },
    },
}
